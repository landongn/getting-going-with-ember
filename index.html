- jQuery plugins made life easy to pump tons of functionality into your apps, but was messy
- backbone made some modest improvements in the lifecycle by introducing routing
- Angular was so fancy! Dependency injections, more intelligent routing, data binding!
- Angular was also a mess, in my opinion. The lack of overall vision and project definition felt like module-mud-pie
- Ember went through a ton of churn early, but the focus on routing and being built with more conventions was pleasent
- Ember feels like Objective-C, in a lot of ways. the MVC paradigm has survived since Objective-J / Sproutcore days, this is a good thing

The Basic Pitch

Flexible, Declarative, Composable API.

Mixins, forward-compatable web components.

(if you use ember-cli) write ES6 JS today.

Dedication to sane depreciations, RFC process.

Constantly Improving.

Behavioral actions API.

Writing ember fast makes you feel like a superhero.


- No focus on build tools, no focus on compilers, no focus on testing integration. It's the most batteries included framework with ember-cli
- jQuery, Handlebars (Htmlbars), RSVP, Babel, node, etc
- ES6 opens up doors for proper modularization, getting rid of quirks and tribulations of things like r.js, browserify,

export default Ember.Component.extend(SomeMixin, {
  // creates a section tag
  tagName: '',

  // will populate the class attr of the tag
  classNames: ['rad-component'],

  // binds dataColumns to data-colums and dataRows to data-rows attrs
  attributeBindings: ['data-columns', 'data-rows'],

  // binds the isActive component property to the class 'is-active'
  // based on isActive being a truthy property
  classNameBindings: ['isActive', 'isDisabled']

Global State

Set data-state="something" on a slide and "something" will be added as a class to the document element when the slide is open. This lets you apply broader style changes, like switching the page background.

State Events

Additionally custom events can be triggered on a per slide basis by binding to the data-state name. 


Reveal.addEventListener( 'customevent', function() {
	console.log( '"customevent" has fired' );
} );

Take a Moment

Press B or . on your keyboard to pause the presentation. This is helpful when you're on stage and want to take distracting slides off the screen.
