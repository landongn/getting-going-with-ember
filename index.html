The Router

The most important element of ember.

Defines states the application can react to.

Example Routing Definition


// app/router.js
Router.map(function() {

  // defines a standalone route with no substates or arguments
  // routable as 'stateA'
  this.route('stateA');

  // defines a standalone route with no substates and an argument
  // routable as 'stateB'
  this.route('stateB', {path: '/stateB/:someArgument'});

  // defines a route with a child state
  // routable as superState.loading, superState.error,
  // superState.index, superState.childState
  this.route('superState', function () {
  	this.route('childState');
  });

});

The Router is kind of like a rubix cube with callbacks.

When the router detects a change, it kicks off a series of ordered events -- a lifecyle -- that reconfigures all the things.

It first configures it's Route classes:

beforeModel(transition)

model(params)

afterModel()

setupController(controller, model)

renderTemplate()

then, once the templates are rendered (in Route::renderTemplate).

controller contexts are created based on what's defined in the templates (more on that later)

view lifecycle events then start:

beforeModel(transition)

willInsertElement()

didInsertElement()

willDestroyElement()

Treat the lifecycle as a collection of $.on() style event handlers that are executed in a specific order.

Going Fast Tip #1:

When building your app, build features by augmenting the lifecycle through observers, computed properties, and actions.

Augmenting the lifecycle

lets look at some examples

Computed Properties give you an easy way to make your own lifecycle events meaningful:

	
//stateA/controller.js [note: uses prototype extension]
export default Ember.Controller.extend({

  // define a computed property that updates
  // a slice of an array when it's filtering key changes.
  arraySlice: function () {

    var thisSlice = [];
    var filter = this.get("filterProp");
    this.get('arrayData').forEach(function (item, iter) {
      if (item.get(filter)) {
        thisSlice.push(item);
      }
    });
    return thisSlice;
  // old style property signal
  }.property('filterProp');
});

the 2.0 way:

	
//stateA/controller.js [note: does NOT use prototype extensions]
// side note: this is the preferred way to do it.
export default Ember.Controller.extend({

  // define a computed property that updates
  // a slice of an array when it's filtering key changes.
  arraySlice: Ember.computed('filterProp', function (key, value) {

    var thisSlice = [];
    var filter = this.get("filterProp");
    this.get('arrayData').forEach(function (item, iter) {
      if (item.get(filter)) {
        thisSlice.push(item);
      }
    });
    return thisSlice;
  }
});
