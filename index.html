Declarative?

a style of building the structure and elements of computer programs, that expresses the logic of a computation without describing its control flow.

Declarative Templates

Templates are where the structure your app is designed, using outlets and components.

User events go down the stack, actions go up the stack.

we can use this to our advantage when we design our component layer, by selectively computing and binding to things further down the stack.

Components don't need complex dom

writing component-of-components is brutally simple

encouraged, especially for advanced UI

reuse and composability rise naturally from this style

Block vs. Isolate Components


// isolate component, has no {{yeild}}-able content.
{{isolate-component SomeBinding=ParentContext.Property}}

						


// block style component, {{yeild}}-able content.

{{#block-component SomeBinding=ParentContext.Property}}
	// put dom here, more components, whatever
{{/block-component}}

Roll your own lifecycles.

Computed Properties provide ways to make your own lifecycle events meaningful

	
//stateA/controller.js [note: does NOT use prototype extensions]
// side note: this is the preferred way to do it.
export default Ember.Component.extend({

  // define a computed property that updates
  // a slice of an array when it's filtering key changes.
  arraySlice: Ember.computed('filterProp', function (key, value) {

    var thisSlice = [];
    var filter = this.get("filterProp");
    this.get('arrayData').forEach(function (item, iter) {
      if (item.get(filter)) {
        thisSlice.push(item);
      }
    });
    return thisSlice;
  }
});
